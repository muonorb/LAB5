# 🎨 Image Mosaic Generator

An interactive photomosaic generator that reconstructs any image using small colored tiles. Built with Python, NumPy, and Gradio — featuring a fully vectorized pipeline that achieves **~50× speedup** over the naive loop-based implementation.

---

## 📋 Overview

The app takes any input image, divides it into a grid of cells, and replaces each cell with the visually closest tile from a pre-built tile library of 87 colored tiles. The result is a mosaic that resembles the original image when viewed from a distance.

**Core philosophy**: First make it work → then make it fast → then make it maintainable.

## 🚀 Usage

### Launch the Gradio app

```bash
# Auto-reloads on file save (recommended during development)
gradio app.py

# Or single run
python app.py
```

Then open **http://localhost:7860** in your browser.

### Using the app

1. Upload any photo using the image upload box
2. Adjust the **Grid Cells** slider (8 to 64) — smaller = more detail
3. Click **Generate Mosaic**
4. View the mosaic output and stats:
   - Grid size and output resolution
   - Processing time
   - MSE (lower = more similar to original)
   - SSIM (higher = more similar, max 1.0)

### Python API

```python
from mosaic_generator import MosaicBuilder, TileManager

# Initialize
tile_manager = TileManager(tile_directory='tiles/', tile_px=32)
tile_manager.load_tiles()
builder = MosaicBuilder(tile_manager, grid_size=(32, 32))

# Generate mosaic
mosaic = builder.create_mosaic(input_image)
metrics = builder.compute_similarity(input_image, mosaic)
print(metrics)  # {'mse': 54.84, 'ssim': 0.9930}
```

## ⚡ Performance

### Benchmark Results

All benchmarks run on random 256×256, 512×512, and 1024×1024 images. Part 1 = naive nested loop implementation. Part 2 = fully vectorized implementation.

| Image Size | Grid Size | Part 1 — Naive (s) | Part 2 — Optimized (s) | Speedup |
|---|---|---|---|---|
| 256×256   | 16×16 | 0.0335 | 0.0007 | 47.2× |
| 256×256   | 32×32 | 0.1280 | 0.0024 | 53.5× |
| 512×512   | 16×16 | 0.0328 | 0.0005 | 59.8× |
| 512×512   | 32×32 | 0.1271 | 0.0021 | 59.4× |
| 1024×1024 | 32×32 | 0.1263 | 0.0022 | 57.2× |
| 1024×1024 | 64×64 | 0.4983 | 0.0112 | 44.6× |

Visual quality is maintained: MSE between naive and optimized outputs = **54.84**, SSIM = **0.9930**.

### Key Optimizations

1. **Vectorized grid extraction** — NumPy `reshape + transpose` replaces nested Python loops over all grid cells
2. **BLAS matrix multiply for color matching** — computes all N_cells × N_tiles distances in one `@` operation instead of 89,088 Python iterations
3. **`cv2.resize(INTER_AREA)` for cell averaging** — C++ block averaging replaces 1,024 per-cell NumPy mean calls
4. **Tile caching** — 87 tiles loaded once at startup, never re-read from disk
5. **Pre-computed `||tile||²`** — stored in `__init__`, reused on every mosaic call
